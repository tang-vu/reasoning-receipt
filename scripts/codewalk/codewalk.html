1 — Circle developer-controlled Wallets

Provisioning + USDC transfers

wallets/circle.py
CIRCLE_BASE = "https://api.circle.com/v1/w3s"

def transfer_usdc(self, *, from_wallet_id, to_address, amount_usdc) -> str:
    payload = {
        "idempotencyKey": str(uuid.uuid4()),
        "entitySecretCiphertext": self.entity_secret,   # RSA-OAEP encrypted
        "amounts": [str(round(amount_usdc, 6))],  # USDC = 6 decimals
        "destinationAddress": to_address,
        "tokenId": os.getenv("CIRCLE_USDC_TOKEN_ID"),
        "walletId": from_wallet_id,
    }
    resp = client.post(f"{CIRCLE_BASE}/developer/transactions/transfer",
                       headers=self._headers(), json=payload)
Two developer-controlled wallets — a portfolio wallet (holds USDC, places trades, receives paywall fees) and a consumer wallet (pays the oracle for its own queries). Provisioned headlessly in ~4s via scripts/circle-setup.py: RSA-OAEP-encrypt the entity secret, register it, create a walletSet, create the two wallets — zero console clicks.
2 — Gateway / x402 v2 paywall

Per-call USDC monetization, EIP-3009 signed

server/x402.py
def challenge_body(self, challenge) -> dict:
    # The Circle x402-v2 body returned in the 402 PAYMENT-REQUIRED header
    return {
        "x402Version": 2,
        "accepts": [{
            "scheme": "exact",
            "network": "eip155:5042002",        # CAIP-2 — Arc Testnet
            "asset":   "0x3600...0000",            # USDC on Arc
            "amount":  "10000",                   # 0.01 USDC (6dp)
            "payTo":   challenge.receiver,        # portfolio wallet
            "extra": {
                "name": "GatewayWalletBatched",    # EIP-712 domain
                "verifyingContract": "0x0077777d7EBA46...19B9",
            },
        }],
    }
    # client signs EIP-3009 TransferWithAuthorization -> X-PAYMENT header
    # server forwards to gateway-api-testnet.circle.com/v1/settle
Drop-in per-request paywall for any FastAPI route or MCP tool. The server issues a spec-compliant 402 challenge, the consumer signs an EIP-3009 authorization, and settlement runs through Circle's Gateway facilitator — gasless USDC transfer on Arc. Built for agents paying agents.
